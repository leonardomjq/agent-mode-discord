<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time → runtime | esbuild output becomes the loaded extension; a malicious dep could smuggle code |
| manifest → VS Code | `package.json` declares activation events + capabilities that VS Code enforces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Tampering | package.json manifest / src/**/*.ts | mitigate | Scaffold never introduces `enabledApiProposals` or `(vscode as any)` patterns. Plan 01-03 adds the CI grep guard that enforces this in perpetuity. |
| T-01-06 | Elevation of Privilege | capabilities.untrustedWorkspaces / virtualWorkspaces | mitigate | Manifest declares `untrustedWorkspaces: { supported: true }` (extension never executes workspace code) and `virtualWorkspaces: false` (requires local IPC + local git — remote/virtual is out of scope per PRD §8). |
| T-01-07 | Spoofing (supply chain) | runtime deps | mitigate | Only `@xhayper/discord-rpc@^1.3.1` permitted; pnpm `--frozen-lockfile` in CI (added in plan 01-03) rejects drift. |
</threat_model>
