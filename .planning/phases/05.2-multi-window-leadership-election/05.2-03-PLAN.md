    SECTION 2 — Background (rewritten). Explain in 2-3 paragraphs:
      - VS Code can run multiple windows simultaneously, each with its own extension host process.
      - Pre-05.2 behavior was "every window opened its own Discord IPC connection; Discord merged them" — this was inconsistent across Discord versions and confused users.
      - As of 05.2, Agent Mode runs a leadership election: the first window to start writes `~/.claude/agent-mode-discord.leader.lock` (atomic open with the `wx` flag per D-02) and owns Discord presence; subsequent windows detect the lockfile and stay silent (no Discord IPC opened).

    SECTION 3 — The Contract (bullet list, one per decision):
      - First window owns presence (D-02).
      - Heartbeat: leader touches the lockfile mtime every 30 seconds (D-03).
      - Staleness threshold: a follower observing a lockfile mtime older than 90 seconds attempts a greedy stale-takeover (D-04).
      - Followers do not open Discord IPC, do not run detectors, do not consume CPU beyond a 30s poll (D-05).
      - On takeover, the new leader starts fresh from IDLE — no state replay (D-06).
      - On graceful exit, the leader unlinks the lockfile (D-07). On crash, the lockfile becomes stale within 90s (D-08).
      - No PID-liveness check; mtime is the sole signal (D-08).
