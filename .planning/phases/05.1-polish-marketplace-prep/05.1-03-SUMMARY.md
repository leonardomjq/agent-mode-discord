## What Was Built

**Task 1 — D-10 Privacy FAQ rewrite**

Replaced the single "Nothing. It communicates exclusively..." sentence with a two-paragraph explanation that accurately distinguishes:
- Extension code: zero outbound HTTP requests (CI-verified via `scripts/check-no-network.mjs`, pattern list named explicitly)
- Discord IPC: local-only socket to Discord desktop client (intentional, not our code)
- Discord's server hop: controlled by the Discord client, subject to Discord's privacy policy (linked)

**Task 2 — D-08 Bus factor subsection**

Added `### Bus factor — using your own Client ID` as the last H3 under the Privacy FAQ H2, before the `---` separator. Content:
- Names the bundled Client ID literal (`1493599126217297981`) so users can verify it
- Explains the single-account dependency risk with "hit by the proverbial bus" maintainer-posture tone matching CONTRIBUTING.md
- Documents the 4-step override path via `agentMode.clientId` VS Code setting
- Also documents `AGENT_MODE_CLIENT_ID` env var override (cross-links to `src/rpc/client.ts` line 9)
- Notes custom Discord asset uploads require owning the Application

**Task 3 — D-09 Observability section**

Added `## Observability` as a new H2 between Privacy FAQ and Competitive Positioning (confirmed by line order: 157 < 175). Content:
- Explicitly states no extension telemetry ships in v0.1.0
- Documents Discord Developer Portal metrics: DAU/MAU, activity counts, authorized installs
- Notes these are visible only to the Application owner (not public)
- Notes Marketplace + OpenVSX install counts available after Phase 6 publishes
- Uses descriptive-not-promissory wording throughout ("Discord exposes", not "we track/collect/monitor")
