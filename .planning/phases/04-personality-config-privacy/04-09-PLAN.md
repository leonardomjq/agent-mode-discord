    Create `scripts/check-no-network.mjs`:

    ```js
    #!/usr/bin/env node
    import { readFileSync, existsSync } from "node:fs";

    const BUNDLE = "dist/extension.cjs";

    if (!existsSync(BUNDLE)) {
      console.error(`FAIL: ${BUNDLE} not found — run \`pnpm build\` first.`);
      process.exit(1);
    }

    const src = readFileSync(BUNDLE, "utf8");

    // Patterns that indicate outbound HTTP (PRIV-07 violation).
    // Ordered from most-specific to most-general; each is a reason-annotated entry.
    const FORBIDDEN = [
      { re: /\bhttp\.request\b/g,           reason: "node:http.request outbound call" },
      { re: /\bhttps\.request\b/g,          reason: "node:https.request outbound call" },
      { re: /\brequire\(["']undici["']\)/g, reason: "undici HTTP library" },
      { re: /\brequire\(["']node-fetch["']\)/g, reason: "node-fetch" },
      { re: /\bfrom\s+["']undici["']/g,     reason: "undici import" },
      { re: /\bfrom\s+["']node-fetch["']/g, reason: "node-fetch import" },
      { re: /\bnew\s+XMLHttpRequest\b/g,    reason: "XMLHttpRequest" },
      { re: /\bglobalThis\.fetch\b/g,       reason: "global fetch use" },
    ];

    // `fetch(` is intentionally left OFF the forbidden list at full strength because
    // esbuild may emit `fetch(` in dead-code branches of transitive libs. We scan for
    // it separately and require a whitelist justification for every hit.
    const SUSPECT_FETCH = /\bfetch\s*\(/g;

    // Whitelist: patterns that are known-safe (IPC via net.createConnection with path arg).
    // We do NOT whitelist anything in FORBIDDEN — those fail outright.

    let failed = false;

    for (const { re, reason } of FORBIDDEN) {
      const matches = src.match(re);
      if (matches && matches.length > 0) {
        console.error(`FAIL: ${matches.length}× ${reason} (${re}) in ${BUNDLE}`);
        failed = true;
      }
    }

    // Extra hygiene: if the bundle contains suspicious `fetch(` calls, surface them.
    // These are warnings, not failures — but they should be reviewed.
    const fetchMatches = src.match(SUSPECT_FETCH);
    if (fetchMatches && fetchMatches.length > 0) {
      console.warn(`WARN: ${fetchMatches.length}× 'fetch(' call(s) detected. Review manually: may be dead code in a transitive dep.`);
    }

    // Net.createConnection is legitimate for IPC (Discord unix socket / named pipe).
    // Discord IPC uses { path: "/tmp/discord-ipc-0" } or \\?\pipe\discord-ipc-0.
    // A non-IPC use would pass { host, port } — we'd need to distinguish. For v0.1 static
    // grep, we accept all net.createConnection calls since the only runtime lib is
    // @xhayper/discord-rpc which documents IPC-only transport.

    if (failed) {
      console.error(`\nPRIV-07 violation: bundle contains forbidden HTTP surface. See matches above.`);
      process.exit(1);
    }
    console.log(`PASS: zero outbound HTTP patterns in ${BUNDLE} (${src.length} bytes scanned)`);
    ```

    Add to `package.json` scripts: `"check:no-network": "node scripts/check-no-network.mjs"`.

    **Sanity check manually:** run `pnpm build && node scripts/check-no-network.mjs`. Expected output: PASS. If FAIL, inspect the match — if it's a real violation, PRIV-07 is broken (escalate, don't whitelist). If it's a false positive from `@xhayper/discord-rpc` internals, add a narrow reason comment above the whitelist but DO NOT extend FORBIDDEN whitelist — instead, sharpen the regex (e.g. `/\bhttp\.request\s*\(/` to require a call).

<pitfalls>
- Do NOT whitelist an HTTP pattern by adding it to FORBIDDEN's inverse. If a forbidden pattern appears, SHARPEN the regex (match `\bhttp\.request\s*\(` instead of `\bhttp\.request\b`) or remove the offending import from source. Never loosen the guard.
- Do NOT add `check:no-network` to the pre-commit hook — it requires a built bundle which slows commits. PR-time CI is the right gate.
- Do NOT try to distinguish Discord IPC's `net.createConnection({path})` from HTTP `net.createConnection({host,port})` via regex — too brittle. The static scan blocks the *import* of HTTP libs; IPC via `net` stays untouched.
- The runtime network intercept (Option 1 full from 04-RESEARCH.md §Pattern 7) is NOT implemented here. Document this explicitly in the plan SUMMARY: "v0.2 may add @vscode/test-electron + monkey-patched http for 10-minute nightly runtime assertion. Static grep is the v0.1 gate."
</pitfalls>
